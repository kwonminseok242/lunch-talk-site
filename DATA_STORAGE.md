# 📦 데이터 저장 위치 설명

## 현재 저장 방식

### 로컬 실행 시
- **저장 위치**: `questions.json` 파일
- **위치**: 프로젝트 폴더 내 (`/lunch-talk-questions/questions.json`)
- **특징**: 파일이 로컬에 저장되어 영구 보관됨

### Streamlit Cloud 실행 시
- **저장 위치**: 각 앱 인스턴스의 임시 파일 시스템
- **문제점**: 
  - 앱이 재시작되면 데이터가 사라질 수 있음
  - 여러 사용자가 동시에 접속하면 각각 다른 인스턴스에서 실행되어 데이터가 공유되지 않을 수 있음

## ⚠️ Streamlit Cloud의 제한사항

Streamlit Cloud는 **무상태(stateless)** 앱입니다:
- 각 요청마다 새로운 인스턴스가 생성될 수 있음
- 파일 시스템이 임시적임
- 여러 사용자가 같은 데이터를 공유하기 어려움

## 💡 해결 방법

### 방법 1: Google Sheets 연동 (권장)

모든 사용자가 같은 Google Sheets를 공유하여 데이터를 저장합니다.

**장점:**
- ✅ 모든 사용자가 같은 데이터를 볼 수 있음
- ✅ 데이터가 영구적으로 저장됨
- ✅ Google Sheets에서 직접 확인 가능
- ✅ 설정이 비교적 간단함

**단점:**
- Google 계정 및 API 설정 필요

### 방법 2: 데이터베이스 사용 (Supabase, Firebase 등)

**장점:**
- ✅ 실시간 동기화
- ✅ 확장성 좋음

**단점:**
- 설정이 복잡함
- 무료 플랜 제한 있음

### 방법 3: 현재 방식 유지 (간단한 사용)

**장점:**
- ✅ 추가 설정 불필요
- ✅ 빠르게 사용 가능

**단점:**
- ⚠️ 데이터가 사라질 수 있음
- ⚠️ 여러 사용자 간 데이터 공유 어려움

## 🎯 추천

**빠른 사용이 목적이라면**: 현재 방식으로도 충분합니다. 각 사용자가 자신의 질문을 작성하고 확인할 수 있습니다.

**모든 사용자가 같은 데이터를 봐야 한다면**: Google Sheets 연동을 추가하는 것을 권장합니다.
